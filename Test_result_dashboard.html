Technical Solution for Universal Test Results Dashboard

This project provides a better and deep understanding of the status of our software reliability considering the whole test history, and also a detail overview of the test cases status through all connected test apps.

Project Overview

Objective: Develop a real-time test results dashboard that integrates seamlessly with various testing tools (Tosca, SOAPUI, TestNG, SAP CBTA, and more). It will provide test execution results, logs, and analytics while ensuring modularity and scalability for future integrations.

Key Features

  • Integration with multiple testing tools:
    • Tosca, TestNG, SoapUI, SAP CBTA, and others.
    • Supports both real-time and batch data submission.
  • Centralized results repository:
    • A JSON-based schema to ensure compatibility across tools.
    • Historical data storage and retrieval for trend analysis.
  • Real-time dashboard:
    • Live updates using WebSockets.
    • Detailed test execution data, logs, and metrics.
  • Customizable filters and views:
    • Filter results by tool, environment, date range, and status.

Technical Solution

This technical solution outlines the detailed approach to integrating Tosca and SoapUI real-time reporting with the Universal Test Results Dashboard.

1. Options for Extracting Tosca Results

You can extract and send Tosca test results in real-time using one of the following methods:

  • Tosca CLI + Script: Export test results as XML files and convert them into a format that can be sent to the dashboard API.
  • Tosca Event Listeners: Provides real-time updates during test execution by listening to specific test events.
  • Tosca REST API: Uses the Tosca Server API to manage and extract results, ideal for enterprise-level setups.

2. Integrating SoapUI Test Results

SoapUI test results can also be sent to the dashboard in real-time or through batch processing. Below are the integration methods:

  • SoapUI CLI + Script: Run tests via the SoapUI CLI, export results in XML/JSON, and convert them to the dashboard-compatible JSON schema.
  • Groovy Scripting for Real-Time Reporting: Embed Groovy scripts in SoapUI to send results directly to the dashboard API during test execution.

3. Recommended Solutions

Here’s how each tool can be integrated with the dashboard:

  • Tosca: Export test results via Tosca CLI, process them using scripts, and send them to the API for real-time updates.
  • SoapUI: Use Groovy scripts within SoapUI to automate API submissions as tests execute. Alternatively, use the CLI for batch processing.

5. Dashboard Integration

The dashboard provides the following features:

  • Real-Time Updates: Live results from Tosca and SoapUI are streamed to the dashboard as they execute.
  • Test Details View: Displays test cases with key details, including logs, assertions, and failure reasons.
  • Historical Analysis: Stores all test data in a centralized PostgreSQL database for trend analysis and reporting.

6. Tools & Technologies

  • Frontend: React.js for building the dashboard interface.
  • Backend: FastAPI for handling API requests and real-time WebSocket updates.
  • Database: PostgreSQL with JSONB support for flexible test result storage.
  • Integration: Scripts and APIs for Tosca and SoapUI, with flexibility to integrate other tools like TestNG and SAP CBTA.

Integrating Kiwi TCMS with TestTrack (combination with Test Management Tool)

Kiwi TCMS is an open-source test case management system that provides centralized test repositories, execution tracking, and reporting. Here’s how you can integrate Kiwi TCMS with your TestTrack app:

1. Setting Up Kiwi TCMS

  • Hosting: Set up Kiwi TCMS using Docker for quick deployment or install it manually on your server.
  • Access: Access the web interface at http://localhost:8080 (or your server's IP).
  • Database: Connect Kiwi TCMS to PostgreSQL for better scalability.

2. Kiwi TCMS API Configuration

Enable the Kiwi TCMS REST API to allow communication between TestTrack and Kiwi TCMS:

  • Generate API Key: Navigate to Profile > API Keys in Kiwi TCMS to create an API token.
  • API Endpoints: Use the following endpoints to interact with Kiwi TCMS:
    • POST /api/v1/testexecution/ to send test execution results.
    • POST /api/v1/testcase/ to import test cases.

3. Connecting Kiwi TCMS with TestTrack

To push test results from TestTrack to Kiwi TCMS:

  • Send Test Results: Configure TestTrack to send JSON payloads to the Kiwi TCMS API endpoint after test execution.
  • Sync Test Cases: Periodically synchronize test cases between Kiwi TCMS and TestTrack using the API.
  • Real-Time Updates: Use Kiwi TCMS Webhooks to notify TestTrack of new test results or updates.

  • Improved visibility into software reliability and test case coverage.
  • Scalable and customizable for future tools and workflows.

  • API Communication: Use Python's requests library or JavaScript's axios for integration.
  • Webhooks: Automate real-time updates from Kiwi TCMS to TestTrack.
